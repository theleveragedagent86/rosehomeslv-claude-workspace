THE FOLDER PATTERN (DO THIS BEFORE YOU RUN IT)
━━━━━━━━━━━━━━━━━━━━
Cowork is only as smart as the folder you point it at. Before you run the slash command:

1. Make a folder named after the listing address (`29 Amber Rock Street`)
2. Drop in everything you've got — Zillow URL, listing package PDF, MLS data sheet, photos, seller's notes
3. Open Cowork inside that folder as your active project
4. Then run `/listing-description [address]`

Claude reads everything in the folder before it writes a single word. Garbage in → generic out. Real inputs in → publication-ready out.

━━━━━━━━━━━━━━━━━━━━
THE TWO EDITS I MAKE EVERY TIME
━━━━━━━━━━━━━━━━━━━━
1. **Cut anything you can't defend.** If it says "walk-in pantry" and the pantry is actually 36 inches deep, cut it. AI doesn't know your property — you do. Don't sign your name to a claim you can't back up.
2. **Swap dimensions for descriptors.** "20 by 11 family room" → "oversized family room." "14 by 12 primary suite" → "primary suite with room for a king-size set." Buyers don't measure — they imagine.

That's it. The plug-in does the heavy lifting. Your two-minute edit makes it sound like you, not a robot.
